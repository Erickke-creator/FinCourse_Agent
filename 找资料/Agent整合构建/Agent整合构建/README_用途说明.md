## 二、ML训练数据库 — 用途详解

### 定位
为未来机器学习模型训练提供统一schema的标准化数据集。当前合并了两个来源：
- UCI South German Credit（1000行）：德国历史个人信贷，21个原始变量 → 用于基础二分类教学
- CUMCM 2020 Synthetic（8000行）：基于中国小微企业统计分布生成的合成数据，28个特征 → 用于贴近真实场景的训练

### 核心文件

#### 1. 训练数据_合并版.csv（最重要）
- 行数：9000（1000 UCI + 8000 Synthetic）
- 列数：49 个统一字段
- 标签列：risk_label（1=违约/0=正常），has_defaulted，credit_rating
- 关键设计：UCI和Synthetic各自独有的字段在另一方中为空白（不填0伪装）
- 使用方式：按 dataset 列筛选 → UCI_South_German_Credit 或 CUMCM_2020_Synthetic

#### 2. 特征工程方案.md
- 特征分为 A~G 七大类，每类标注 ML 重要性（critical/major/moderate）
- 标签工程公式（基于山东大学学报2024 AUC=0.964模型系数）
- 预处理流水线：缺失值处理 → 类别编码 → StandardScaler → 模型训练

#### 3. 字段字典_统一版.csv
- 49个字段的中文名称、数据类型、来源标注（UCI/Synthetic）、Agent可用性
- 区分source（原始公开字段）和synthetic（合成/模拟字段）

#### 4. 模型训练参数与评估.json
- 当前3模型：XGBoost违约预测 / GradientBoosting信用评级 / RandomForest风险分类
- XGBoost AUC=0.701, Accuracy=0.631
- Top-10特征重要度：overdue_count_2yr 最重要（权重17%）
- ⚠️ 改进方向：使用 CUMCM 2020 真实数据重训 → AUC可达0.90+

## 三、Agent知识库 — 用途详解

### 定位
Agent 在运行时的可检索知识层，不进入ML模型训练。用于 RAG 检索、规则校验、产品匹配和解释生成。

### Agent 调用流程
```
用户输入经营数据
    |
    +--> [ML模型] 调用训练数据库 --> 输出违约概率/信用评级/风险等级
    |
    +--> [规则引擎] 检索知识库 --> 逐条命中规则诊断
    |     +-- 政策规则知识库.csv       --> 监管合规检查
    |     +-- 行业准入与限制规则.csv    --> 行业准入判断
    |     +-- 征信要求分级表.csv        --> 征信诊断
    |     +-- 纳税等级评分规则.csv      --> 银税贷匹配
    |     +-- 贷款被拒因子与权重.csv    --> 风险归因
    |
    +--> [银行匹配引擎] 检索知识库 --> 排序推荐
    |     +-- 银行产品数据库.json       --> 28家银行产品匹配
    |
    +--> [案例匹配] 检索知识库 --> 相似情景提示
    |     +-- 教学案例库.csv            --> 30个情景案例
    |
    +--> [政策补贴检查] 检索知识库
    |     +-- 2026政策补贴与贴息规则.csv --> 贴息/担保资格
    |
    +--> [综合输出] 融资诊断 + 改善建议 + 银行推荐 + 材料清单
```

### 各文件详解

#### 1. 政策规则知识库.csv（18条规则）
- RULE-001~002：金融监管总局·小微金融监管评价办法 → 政策背景解释
- RULE-003~004：国务院·普惠金融实施意见 → 普惠定位+信用增级建议
- RULE-005~007：金融监管总局·流动资金贷款办法 → 材料清单+用途校验+额度建议
- RULE-008~009：金融监管总局·续贷通知 → 续贷资格预诊断
- RULE-010：金融监管总局·尽职免责通知 → 术语解释（防误导）
- RULE-011~012：互联网贷款管理办法 → 线上审批解释+数据合规
- RULE-013~014：中国货币网·LPR → 利率合理性参照
- RULE-015~016：最高法·担保制度解释 → 担保有效性提示
- RULE-017~018：中国银行·中银企E贷 → 产品初筛+材料清单
- Agent检索方式：按 rule_theme 或 agent_usage 关键词匹配 → 返回 key_condition + risk_warning + source_url

#### 2. 银行产品数据库.json（28家银行）
- 覆盖：6国有+7股份+8城商+3互联网+2外资
- 每条含：产品名/贷款类型/最高额度/利率区间/最长期限/准入条件/偏好权重/被拒敏感度/目标企业/估计通过率
- 附：贷款被拒因子、禁止行业清单、各层级征信要求、ML特征定义、2026政策补贴、宏观统计
- Agent匹配逻辑（参照 bank_engine.py calculate_bank_approval 函数）：
  通过概率 = 银行基准通过率 x 经营年限因子 x 抵押要求因子 x 征信因子 x 行业接受度 x 偏好匹配 x 健康度因子
  利率 = 银行最低利率 + 逾期惩罚 - 房产优惠
  额度 = min(银行最高额度, 月营收x0.4x12, 房产价值x0.65)

#### 3. 教学案例库.csv（30条）
- 7种案例类型各3条基础+3条变体：获批/拒绝/逾期/违约/担保纠纷/续贷/供应链
- 全部标注 is_synthetic=1，不冒充真实案例
- Agent用途：按 case_type 或 risk_signal 匹配 → 返回相似情景的诊断思路

#### 4. 行业准入与限制规则.csv
- 18个行业的准入等级（正常/一般/谨慎/严格限制）和接受度系数（1.0 ~ 0.30）
- Agent用途：输入行业类型 → 返回准入等级+接受度系数 → 参与银行匹配概率计算

#### 5. 征信要求分级表.csv
- 国有大行/股份制/互联网/外资四级的逾期容忍度、征信查询上限、五级分类处理规则

#### 6. 贷款被拒因子与权重.csv
- 7大被拒原因及占比：征信不良35%/经营流水不足25%/缺乏抵押物20%/成立时间短10%/财务不健全5%/行业限制3%/负债率高2%

#### 7. 2026政策补贴与贴息规则.csv
- 6类政策：服务业纾困/创业担保/绿色金融/数字产业/三农专项/科创企业
- Agent用途：根据企业行业+科创属性 → 检索匹配政策补贴 → 输出贴息/担保/额度信息

#### 8. 宏观统计数据.json
- 普惠小微贷款余额32.93万亿元（2024年末）/同比增长14.6%/四大行排名
- Agent用途：报告开头的宏观背景引用

#### 9. 纳税等级评分规则.csv
- A=5/B=4/M=3/C=2/D=1 五级评分
- Agent用途：银税贷产品匹配 + 健康度评分中的税务维度

#### 10. 数据来源登记表.csv + 数据语义层与口径说明.md
- risk_label=1 = 坏账/违约
- is_synthetic=1 = 教学模拟案例（非真实）
- 空白字段 = 公开源不存在该字段（不等于0）
- 所有输出仅用于课程展示，不构成真实授信审批或法律意见

## 四、边界与免责

| 边界 | 说明 |
|------|------|
| 模型数据 | UCI为德国个人信贷，Synthetic为统计模拟，均非真实中国银行小微企业授信数据 |
| 案例数据 | 30条全部为合成教学案例，is_synthetic=1，非真实客户或裁判案例 |
| 政策资料 | 公开资料整理摘要，非官方全文或法律意见 |
| 银行产品 | 基于公开信息整理，产品参数可能变化，以银行最新页面和审批为准 |
| 模型输出 | 仅用于课程展示/教学演示，不能作为真实授信审批依据 |
| 法律风险 | Agent只做风险提示和材料建议，不做法律结论；法律争议建议咨询专业机构 |
