|------|------|
| **产品类型** | 纯信用（可叠加抵押提额） |
| **最高额度** | 500万元（信用最高300万） |

## 4. 城市/农村商业银行贷款产品

### 4.1 城商行典型特点

城商行（125家）因地域差异较大，总体特点：
- **利率**：一般高于国有大行100-200BP，约3.5%-6%
- **额度**：一般在50-300万之间
- **本地化优势**：了解当地企业经营状况，审批更灵活
- **政策倾斜**：大量使用地方政府增信和风险分担机制

### 4.2 农商行典型产品

| 产品类型 | 特点 | 额度 | 放款速度 |
|----------|------|------|----------|
| **精微快贷** (侯马农商行) | 纯信用、无抵押、手机端申请 | 灵活 | 分钟级 |
| **苏信贷** (沛县农商行) | 全流程线上化 | — | 最快2小时 |
| **创业担保贷** (利辛/淅川农商行) | 政府贴息、退役军人/大学生 | 20-50万 | 秒批秒贷 |
| **整村授信** (河南方城模式) | 行政村全覆盖 | — | — |

### 4.3 农商行典型申请条件

| 条件维度 | 要求 |
|----------|------|
| **主体资格** | 农户、个体工商户、小微企业主、返乡创业者 |
| **经营要求** | 有稳定经营流水/项目 |
| **信用要求** | 信用记录良好，无重大不良 |
| **担保方式** | 纯信用为主，部分大额需省农担增信 |
| **年龄** | 一般18-65周岁 |

| | 地区经济水平 | 连续 | ★★★ |

### 6.2 银行审批的5C原则对应指标

| 5C维度 | 对应可量化指标 |
|--------|--------------|
| **Character（品格）** | 法人征信、企业征信、纳税等级、法律纠纷 |
| **Capacity（能力）** | 经营流水、利润率、现金流、行业经验 |
| **Capital（资本）** | 实收资本、所有者权益、资产负债率 |
| **Collateral（抵押）** | 固定资产价值、房产区域、抵押率 |
| **Condition（环境）** | 行业前景、地区GDP、政策支持 |

## 7. 贷款被拒原因与统计分析

### 7.1 主要被拒原因（按频率排序）

| 排名 | 被拒原因 | 估计占比 | 说明 |
|------|----------|----------|------|
| 1 | **征信记录不良** | ~35% | 法人或企业征信有逾期、呆账、关注类记录 |
| 2 | **经营流水不足** | ~25% | 银行流水不足以证明还款能力 |
| 3 | **缺乏有效抵押物** | ~20% | 无固定资产或抵押物价值不足 |
| 4 | **企业成立时间太短** | ~10% | 经营不满1年，缺乏经营历史 |
| 5 | **财务制度不健全** | ~5% | 无规范财务报表、发票管理混乱 |
| 6 | **行业限制** | ~3% | 属于银行禁止或限制准入行业 |
| 7 | **负债率过高** | ~2% | 已有过多贷款，资产负债率超标 |

### 7.2 银行禁止/限制准入行业（负面清单）

| 类型 | 具体行业 |
|------|----------|
| **禁止准入** | 赌博、色情、毒品、非法集资、传销等非法行业 |
| **严格限制** | 房地产开发（非自用）、产能过剩行业（钢铁、水泥、电解铝等）、高污染行业 |
| **审慎准入** | 娱乐行业（KTV、酒吧等）、投资理财类、煤炭开采 |
| **有条件准入** | 建筑业（需资质）、贸易类（需真实交易背景） |

### 7.3 银行对征信的具体要求（典型标准）

| 征信项目 | 国有大行要求 | 股份行/中小银行要求 |
|----------|------------|-----------------|
| 近2年逾期次数 | ≤3次（建行≤6次） | ≤6次 |
| 近3个月征信查询 | ≤3-4次 | ≤6次 |
| 当前逾期 | 必须结清 | 必须结清 |
| 五级分类 | 正常类 | 正常类 |
| 关注类 | 一般拒绝 | 视情况而定 |
| 次级/可疑/损失 | 直接拒绝 | 直接拒绝 |
| 对外担保 | 纳入负债考虑 | 纳入负债考虑 |

## 10. 系统开发建议

### 10.1 推荐模型架构

```
用户输入层 → 特征工程层 → 多模型评分层 → 银行匹配层 → 结果输出层
```

#### 输入层（小微企业自填 + 可选项）
1. **基本信息**：企业名称、统一社会信用代码、注册时间、注册资本
2. **经营数据**：近12个月营收、净利润、经营流水
3. **税务数据**：纳税等级、近2年纳税总额
4. **征信数据**：法人/企业征信是否有逾期
5. **资产信息**：是否有房产、房产估值
6. **行业分类**：所属行业
7. **所在地区**：省市

#### 特征工程层
- 参考第6节的指标体系，构建至少20+核心特征
- 特别关注：废票率、交易波动系数、上下游企业数量

#### 模型层（三阶段）
1. **违约概率预测**（XGBoost/LightGBM）：输出企业违约概率
2. **信用评级分类**（Random Forest）：输出A/B/C/D评级
3. **银行匹配推荐**（多分类+排序）：输出每家银行的通过概率

#### 银行匹配层
基于以下规则+ML模型：
- 如果企业有优质房产 → 推荐中信/招行/交行（利率2.15%-2.35%）
- 如果企业无抵押但经营流水好 → 推荐平安/建行/邮储
- 如果企业是电商 → 推荐网商银行
- 如果企业成立时间短 → 推荐邮储/平安/网商银行
- 如果企业纳税好 → 推荐中行银税贷/建行云税贷

### 10.2 建议的特征变量（用于ML训练）

```python
# ============================================
# 第一类：基础特征（essential）
# ============================================
- business_age_years: 企业经营年限
- registered_capital: 注册资本（万元）
- industry_type: 行业分类（one-hot编码）
- region: 所在地区
- employee_count: 员工数量

# ============================================
# 第二类：财务特征（important）
# ============================================
- annual_revenue: 年营业收入（万元）
- annual_profit: 年净利润（万元）
- profit_margin: 利润率（0-1）
- cash_flow: 经营现金流（万元）
- asset_liability_ratio: 资产负债率（0-1）

# ============================================
# 第三类：税务特征（important）
# ============================================
- tax_level: 纳税等级（A=5, B=4, M=3, C=2, D=1）
- annual_tax: 年纳税额（万元）
- invalid_invoice_ratio: 废票率（0-1）

# ============================================
# 第四类：信用特征（critical）
# ============================================
- has_default_history: 历史违约（0/1）
- overdue_count_2yr: 近2年逾期次数
- credit_inquiry_3m: 近3月征信查询次数
- legal_disputes: 法律纠纷数量

# ============================================
# 第五类：资产特征（加分项）
# ============================================
- has_real_estate: 有房产（0/1）
- real_estate_value: 房产估值（万元）
- has_other_collateral: 有其他抵押物（0/1）

# ============================================
# 第六类：经营稳定性特征
# ============================================
- revenue_volatility: 收入波动系数
- customer_count: 上下游客户/供应商数量
- customer_concentration: 客户集中度
```

### 10.3 各银行通过概率预测模型

可构建一个多输出的分类模型，对每家银行输出：
1. **通过概率**（0-1连续值）
2. **预期利率区间**
3. **推荐额度区间**
4. **匹配度评分**

训练数据需要标注每家银行对不同类型企业的审批结果。可以从以下途径获取：
- CUMCM 2020数据（含信用评级）
- 各银行年报中的小微贷款数据
- 行业研报中的银行偏好分析
- 网络爬取的银行产品准入条件（用于规则引擎）

### 10.4 网页端技术栈推荐

| 层级 | 推荐技术 |
|------|----------|
| **前端** | React/Vue + Ant Design/Element UI |
| **后端** | Python Flask/FastAPI |
| **ML模型** | scikit-learn + XGBoost + LightGBM |
| **模型部署** | ONNX Runtime 或直接 pickle 加载 |
| **数据库** | SQLite（轻量）或 PostgreSQL |
| **可视化** | ECharts（银行贷款对比雷达图） |

### 10.5 核心功能页面设计

1. **企业信息录入页**：表单式输入企业各项指标
2. **风险评估结果页**：信用评分仪表盘+违约概率
3. **银行匹配页**：按通过概率排序的银行列表+雷达图对比
4. **贷款方案详情页**：各银行推荐额度、利率、期限对比
5. **改善建议页**：针对被拒风险的改进建议（如"优化纳税记录可提高建行通过率15%"）
